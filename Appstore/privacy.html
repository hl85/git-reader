Privacy Policy

Last Updated: June 26, 2026

Gits Reader ("the App") is a read-only Git repository and Markdown note reader for iOS. We are committed to protecting your privacy. This policy explains what data we handle and how.

1. Data We Store Locally on Your Device

The App stores the following data only on your device and never transmits it to any external server operated by us:

  • Account Metadata: Your Git platform username, platform type (GitHub/GitLab), and server URL are stored in the device's local preferences (UserDefaults) to display your account list.
  • Access Tokens: Personal Access Tokens (PAT) and OAuth tokens are stored securely in the iOS Keychain with device-only access. They are never logged, cached externally, or sent to any third party.
  • Repository Configuration: Repository URLs, branch names, and associated account IDs are stored locally to manage your repository list.
  • App Preferences: Theme selection, language preference, font size, shallow clone depth, and metadata templates are stored locally.
  • Cloned Repository Data: Git repositories are cloned to your device's local sandbox (Documents directory) for offline reading.

2. Network Communications

The App communicates only with the Git servers you configure (e.g., GitHub, GitLab, or your self-hosted Git server). These communications include:

  • Git Operations: Clone, fetch, and push operations using the libgit2 library, connecting directly to your configured repository URLs.
  • Authentication: OAuth Device Flow authentication with GitHub or GitLab APIs (only when you initiate a login).
  • Connection Tests: Lightweight HTTP requests to your repository's /info/refs endpoint to verify connectivity and permissions.

We do not operate any intermediary servers, analytics endpoints, or tracking services. All network traffic goes directly between your device and your chosen Git provider.

3. Third-Party Services

The App does not integrate any third-party analytics, advertising, or tracking SDKs. The open-source libraries used are:

  • swift-libgit2: Local Git operations (no telemetry).
  • swift-markdown: Markdown parsing (Apple open-source, no network).
  • Yams: YAML parsing (no network).
  • Nuke: Image loading and caching (loads images from your own repository).
  • Highlightr: Code syntax highlighting (entirely local).

4. Data Collection

We do not collect, transmit, or store any personal data on external servers. The App has no backend infrastructure. All your data remains on your device.

5. Account Deletion

You can remove your account from the App at any time via the Account Management screen. This will:

  • Delete your access token from the iOS Keychain.
  • Remove your account metadata from local storage.
  • Disassociate any repositories linked to that account.
  • Delete local cloned repository data associated with that account.

To fully revoke the App's access to your Git platform account, you should also visit your platform's application settings:

  • GitHub: github.com/settings/applications
  • GitLab: Navigate to Settings > Applications on your GitLab instance.

6. Encryption

The App uses standard HTTPS/TLS encryption for all network communications, provided by the iOS system's Security framework. No custom or non-exempt encryption is implemented.

7. Children's Privacy

The App does not knowingly collect any data from anyone, including children under the age of 13.

8. Changes to This Policy

We may update this Privacy Policy from time to time. Any changes will be reflected on this page with an updated revision date.

Gits Reader is an independent application and is not affiliated with GitHub, GitLab, or Obsidian.
